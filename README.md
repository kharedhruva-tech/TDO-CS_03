# TDO-CS_03
Conducted a WordPress security assessment using WPScan to identify exposed plugins, outdated components, and publicly disclosed vulnerabilities. The scan revealed WordPress version information, installed plugins, and several known CVEs affecting outdated WordPress core and plugin versions.
Key findings included:

Detection of outdated WordPress core versions with publicly known vulnerabilities.
Identification of vulnerable plugins such as ElementsKit and Akismet.
Discovery of security issues including Stored Cross-Site Scripting (XSS), Sensitive Data Disclosure, and Missing Authorization vulnerabilities.
Enumeration of installed plugins, themes, and security-related HTTP headers.

This exercise enhanced my understanding of WordPress security testing, vulnerability assessment, CVE analysis, and remediation recommendations. All testing was performed for educational and authorized security research purposes within ethical hacking guidelines.

Tools Used: WPScan, Kali Linux, CVE Database, WordPress Security Advisories
